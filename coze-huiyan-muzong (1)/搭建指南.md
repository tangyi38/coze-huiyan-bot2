# 慧眼牧踪 × 扣子(Coze) 搭建指南

## 一、项目结构说明

```
coze-huiyan-muzong/
├── Dockerfile          ← Railway 构建用（解决 better-sqlite3 原生编译）
├── .dockerignore
├── railway.toml        ← Railway 部署配置
├── package.json
├── server.js           ← 主入口
├── db/init.js          ← 数据库初始化（建表+种子数据）
├── middleware/coze-auth.js  ← Coze API Key 双认证中间件
├── routes/
│   ├── auth.js              ← 登录（JWT）
│   ├── sheep-coze.js        ← 羊只管理
│   ├── location-coze.js     ← 定位追踪
│   ├── health-coze.js       ← 健康监测
│   ├── detect-coze.js       ← AI识别
│   ├── alert-coze.js        ← 异常预警
│   ├── trace-coze.js        ← 产品溯源
│   └── settings-coze.js     ← 系统设置
├── ui/                  ← 前端界面
├── openapi.yaml         ← Coze 插件导入用
└── 知识库/              ← Coze 知识库文件
    ├── 羊只养殖技术手册.md
    └── 系统使用指南.md
```

## 二、3 步上线

### 第 1 步：部署后端到 Railway

1. **解压代码包**
   ```bash
   tar xzf coze-huiyan-muzong.tar.gz -d coze-huiyan-muzong
   cd coze-huiyan-muzong
   ```

2. **推到 GitHub**
   ```bash
   git init
   git add .
   git commit -m "init: 慧眼牧踪 Coze 适配版"
   git remote add origin https://github.com/你的用户名/coze-huiyan-muzong.git
   git push -u origin main
   ```

3. **Railway 部署**
   - 打开 https://railway.app ，用 GitHub 登录
   - 点 **New Project** → **Deploy from GitHub repo** → 选你的仓库
   - Railway 会自动检测 Dockerfile 并构建
   - 等待构建完成（约 2-3 分钟）

4. **获取公网地址**
   - 在 Railway 项目 Settings → Networking → 点 **Generate Domain**
   - 得到类似 `coze-huiyan-muzong-production.up.railway.app` 的地址
   - 浏览器访问该地址，应能看到登录页

5. **（可选）设置环境变量**
   - 在 Railway 的 Variables 标签中可设置：
     - `COZE_API_KEY`：Coze 插件认证密钥（默认 `coze-huiyan-muzong-2026`）
     - `JWT_SECRET`：JWT 签名密钥（默认 `huiyan-muzong-secret-2026`）

### 第 2 步：在扣子创建智能养殖助手 Agent

1. **打开** https://www.coze.cn → 创建 Bot

2. **基础设置**
   - 名称：`慧眼牧踪智能养殖助手`
   - 描述：`基于慧眼牧踪智能养殖系统的AI助手，支持羊只查询、健康监测、预警管理、产品溯源等功能`
   - 模型：选 `Doubao-1.5-pro` 或其他模型

3. **添加插件**
   - 点 **添加插件** → **创建插件**
   - 插件名称：`慧眼牧踪API`
   - 创建方式：**通过 OpenAPI 导入**
   - 把 `openapi.yaml` 的内容粘贴进去
   - **修改 `servers.url`**：把 `https://你的公网域名` 替换成 Railway 给你的实际地址（如 `https://coze-huiyan-muzong-production.up.railway.app`）
   - 保存

4. **配置插件认证**
   - 在插件设置中，认证方式选 **API Key**
   - 位置：**Header**
   - 参数名：`Authorization`
   - 值：`Bearer coze-huiyan-muzong-2026`
   - ⚠️ 注意 `Bearer` 和 key 之间有个空格

5. **上传知识库**
   - 在 Bot 的知识库区域，上传两个文件：
     - `羊只养殖技术手册.md`
     - `系统使用指南.md`
   - 分段方式建议：自动

6. **编写人设与 Prompt**

在「人设与回复逻辑」中粘贴以下内容：

```
你是「慧眼牧踪智能养殖助手」，一个专业的畜牧养殖AI助手。你帮助牧场管理者、牧民和消费者查询养殖数据、管理羊群、处理预警、溯源产品。

## 核心能力
1. **数据查询**：通过插件调用慧眼牧踪系统API，查询羊只、羊群、定位、健康、预警、溯源等实时数据
2. **养殖知识**：基于知识库中的养殖技术手册，回答饲养、疾病防治、繁殖、生理指标等问题
3. **系统操作**：帮助用户处理预警、添加记录等系统操作

## 回复规范
- 查询类问题优先调用API获取实时数据，不要凭记忆编造
- 数据类回复用清晰的列表或表格呈现
- 养殖专业问题先查知识库，再结合API数据综合回答
- 如果API调用失败，告知用户当前系统可能暂时不可用
- 保持专业但亲切的语气

## 典型对话示例
用户：查看所有羊群状态
→ 调用 getSheepHerds 接口，返回各羊群概况

用户：有没有需要处理的预警？
→ 调用 getAlertStats 查看统计，再调用 getAlertList(type="越界",status="待处理") 获取详情

用户：羊只#0038的体温正常吗？
→ 调用 getSheepById 查基本信息，再调用 getHealthRecords(sheep_id=38) 查健康记录，结合知识库中羊的正常体温范围(38.5-39.5°C)给出判断

用户：溯源码 HYMZ-2026-A01-0001
→ 调用 getTraceByCode(code="HYMZ-2026-A01-0001") 获取全链溯源信息
```

## 三、常见问题

### Q: Railway 构建失败？
A: 确保代码中包含 Dockerfile。本项目使用 Dockerfile 构建方式（不是 nixpacks），专门解决 `better-sqlite3` 原生模块编译问题。Railway 会自动检测 Dockerfile 并使用它构建。

### Q: 部署成功但访问报错？
A: 检查 Railway 的 Deployments 日志，看 server.js 是否正常启动。确认 `db/init.js` 在构建时已执行（Dockerfile 中 `RUN node db/init.js`）。

### Q: 插件调用返回 401？
A: 检查插件认证配置：
- Header 名称必须是 `Authorization`（不是 `authorization`）
- 值的格式是 `Bearer coze-huiyan-muzong-2026`（Bearer 后有空格）
- Railway 环境变量 `COZE_API_KEY` 如果改过，插件这边也要同步

### Q: 数据丢失了？
A: Railway 的文件系统是临时的，每次重新部署会重置数据库。Dockerfile 中 `RUN node db/init.js` 会在构建时重新初始化种子数据。如需持久化数据，建议改用 Railway 的 PostgreSQL 插件。

### Q: openapi.yaml 中的服务器地址？
A: 必须替换成你 Railway 项目的实际公网地址，格式如 `https://xxx.up.railway.app`，不要带末尾斜杠。
